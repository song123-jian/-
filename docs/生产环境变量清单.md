# 生产环境变量清单

使用 Skill：requirements-doc

## 1. 变量清单

| 变量 | 作用 | 管理端 | 移动端 | 示例 |
| --- | --- | --- | --- | --- |
| `VITE_SUPABASE_URL` | Supabase 项目 URL | 必填 | 必填 | `https://your-project-ref.supabase.co` |
| `VITE_SUPABASE_ANON_KEY` | Supabase publishable anon key | 必填 | 必填 | `your-supabase-anon-key` |
| `VITE_SUPABASE_AUTH_EMAIL_DOMAIN` | 用户名登录转邮箱时使用的域名 | 必填 | 必填 | `inject-erp.example.com` |
| `VITE_SUPABASE_STORAGE_BUCKET` | 文件上传 bucket | 必填 | 必填 | `erp-files` |

## 2. 密钥边界

前端仅允许使用 publishable anon key。以下密钥禁止出现在 `.env.local`、前端部署变量、源码、文档示例和截图中：

- Supabase service role key
- 数据库连接字符串和密码
- 云平台管理 token
- CI/CD 管理密钥
- 任何第三方 API 私钥

## 3. 配置位置

| 环境 | 文件或平台 |
| --- | --- |
| 本地管理端 | `frontend-admin/.env.local` |
| 本地移动端 | `frontend-mobile/.env.local` |
| 生产管理端 | 静态托管平台的管理端项目环境变量 |
| 生产移动端 | 静态托管平台的移动端项目环境变量 |

## 4. 验证方式

1. 启动管理端，打开 `/dashboard`，确认未配置时显示 Supabase 配置提示。
2. 配置变量后重新启动，确认登录接口可连接 Supabase。
3. 打开系统管理/集成中心，确认 Supabase 项目引用与当前生产项目一致。
4. 全仓搜索不得出现 service role key：

```bash
rg -n "service_role|SUPABASE_SERVICE|DATABASE_URL|postgres://|password=" -g "!node_modules" -g "!dist"
```
