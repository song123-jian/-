# 页面治理工作模式

使用 Skill：codebase-orientation、task-tracking、ui-layout-critique、flow-usability-review、frontend-implementation

## 目标

为 `D:\cx\zsc` 建立按页面逐个交付的治理流程。每个页面必须在进入下一页前完成：

1. 页面职责与入口确认
2. 文案、数字、单位、中文一致性检查
3. 布局层级、响应式、空态、错误态检查
4. 关键交互与跨页面跳转检查
5. API 依赖与数据状态检查
6. `npm run check-encoding`
7. `npm run build`
8. 页面级浏览器验证

## 页面顺序策略

按以下优先级推进：

1. 登录与鉴权入口页
2. 默认首页/工作台
3. 主业务高频路径页
4. 其余列表页、配置页、报表页

## 首个交付页

当前锁定首个交付页面为：`frontend-admin/src/views/login/index.vue`

原因：

- 它是所有后台页面的统一入口。
- 它直接影响后续每个功能页的深链验证效率。
- 当前登录成功后仍固定跳转工作台，影响页面级回归效率。
