# 部署说明

使用 Skill：requirements-doc、project-plan、codebase-orientation

## 1. 部署范围

| 部署对象 | 路径 | 产物 |
| --- | --- | --- |
| 管理端 | `frontend-admin` | `frontend-admin/dist` |
| 移动端 | `frontend-mobile` | `frontend-mobile/dist` |
| 数据库 | `database/init.sql`、`database/supabase-cloud.sql` | Supabase PostgreSQL schema、初始数据、RPC、Storage 策略 |

## 2. 前置条件

1. Node.js 18 或更高版本。
2. 已创建 Supabase 项目。
3. 已获取 Supabase Project URL 和 publishable anon key。
4. 已创建或确认 Storage bucket：`erp-files`。
5. 生产域名、HTTPS 证书和静态托管环境已准备。

## 3. 安装依赖

```bash
npm install
npm --prefix frontend-admin install
npm --prefix frontend-mobile install
```

## 4. 数据库初始化

1. 在 Supabase SQL Editor 或 psql 中执行 `database/init.sql`。
2. 再执行 `database/supabase-cloud.sql`。
3. 确认 `erp_login` RPC、`erp-files` bucket 和 Storage 策略存在。
4. 使用管理员账号验证登录和基础资料读取。

详细步骤见 [数据库初始化与迁移说明.md](数据库初始化与迁移说明.md)。

## 5. 环境变量

管理端和移动端生产环境均需配置：

```bash
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
VITE_SUPABASE_AUTH_EMAIL_DOMAIN=inject-erp.example.com
VITE_SUPABASE_STORAGE_BUCKET=erp-files
```

禁止配置到前端的变量：

- Supabase service role key
- 数据库连接密码
- 云平台管理 token
- 任何可绕过 RLS 或后台权限的密钥

## 6. 构建

```bash
npm run build
```

该命令会顺序执行：

```bash
npm --prefix frontend-admin run build
npm --prefix frontend-mobile run build
```

## 7. 发布

1. 将 `frontend-admin/dist` 发布到管理端静态站点。
2. 将 `frontend-mobile/dist` 发布到移动端静态站点。
3. 对 SPA 托管配置 history fallback，将未知路径回退到 `index.html`。
4. 设置 HTTPS、缓存策略和访问域名。
5. 用浏览器验证以下入口：
   - 管理端 `/login`
   - 管理端 `/dashboard`
   - 移动端 `/m/login`
   - 移动端 `/m/home`

## 8. 发布后检查

1. 登录、退出、刷新用户信息。
2. 创建一条基础资料测试数据并删除或禁用。
3. 查询库存和库存台账。
4. 执行采购入库、生产领料、报工、成品入库、销售出库和回款闭环。
5. 打开老板驾驶舱、生产看板、质量看板。
6. 检查系统日志是否记录关键操作。
