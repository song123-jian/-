## 3. 自动化覆盖摘要

当前 `package.json` 聚合执行 39 个测试文件，覆盖基础资料、销售、生产、质量、库存、工资、财务、报表、系统配置、集成中心、云库运维、消息中心、产品化中心、注塑专业 1-13 和移动端关键工具逻辑。

新增 `npm run verify:delivery` 后，交付验证入口已统一为一条命令，便于后续智能体、人工验收和发布前检查复用。

注塑专项新增覆盖：

- `tests/injection-process.test.mjs`：13 模块顺序、工艺卡校验、首件量产门禁、状态动作。
- `tests/injection-site-flow.test.mjs`：齐套检查、安灯异常、配料回料比例、模具保养风险、OEE 计算。
- `tests/injection-traceability.test.mjs`：批次追溯链、标签模板、8D 完整度、MRP 转请购、数据库/API/路由覆盖。
- `tests/mobile-injection.test.mjs`：移动端注塑入口、表单载荷、待办聚合和路由接入。

## 4. 构建警告记录

本次构建成功，但存在以下非阻断警告，建议纳入后续优化：

| 类型 | 位置 | 影响 | 建议 |
| --- | --- | --- | --- |
| Rollup 注释警告 | `node_modules/@vueuse/core/dist/index.js` | 第三方库纯函数注释位置无法解释，不影响构建结果 | 关注依赖版本升级 |
| 浏览器兼容警告 | `bcryptjs` 引入 `crypto` | Vite 将 Node `crypto` 外部化，可能影响浏览器端密码相关逻辑 | 后续评估是否改为浏览器兼容实现 |
| 包体积警告 | 管理端部分 chunk 超过 500 kB | 首屏加载性能可能受影响 | 后续使用动态导入或 manualChunks 优化 |

## 5. 核心路径结论

详见 [核心路径回归记录.md](核心路径回归记录.md)。当前结论：

- 规则层自动化覆盖较好。
- 截图证据已纳入 23 张。
- 注塑专业 13 个管理端路径已按生产、质量、仓库、报表菜单归位并纳入浏览器抽检范围；13 个移动端路径继续保留在移动端“注塑专业”入口下。
- 登录、生产订单创建、跨表库存联动、移动报工到工资、页面级 E2E 仍需增强。
- 正式上线前需补齐 20 条核心路径真实环境截图或录屏。
