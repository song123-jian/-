# 小型注塑厂综合管理系统（ERP + MES）技术文档

## 1. 项目概述

### 1.1 项目背景
面向员工规模约 10 人的小型注塑厂，打造一套覆盖"老板决策、生产执行、品质管控、销售订单、回款跟踪、计件工资"的一体化管理系统。系统兼顾 ERP（企业资源计划）与 MES（制造执行系统）能力，帮助工厂实现订单到回款的全流程数字化。

### 1.2 目标用户
| 角色 | 人数 | 核心诉求 |
|------|------|----------|
| 老板 / 厂长 | 1~2 | 看经营数据、利润、回款、生产效率 |
| 生产主管 / 领班 | 1~2 | 派工、报工、追踪工单进度、算工资 |
| 操作工 | 5~7 | 扫码报工、查看当班任务、确认产量 |
| 品质人员 | 1 | 巡检、首件/末件检验、不良记录、追溯 |
| 销售 / 跟单 | 1 | 录订单、跟交期、跟回款 |

### 1.3 设计原则
1. **轻量易用**：10 人规模，拒绝复杂审批流，操作三步以内完成。
2. **移动优先**：车间工人以手机/平板扫码为主，办公室以 PC 管理为主。
3. **计件透明**：产量与工资实时挂钩，工人可随时查询。
4. **数据闭环**：销售订单 → 生产工单 → 领料 → 报工 → 质检 → 入库 → 发货 → 回款。
5. **低成本**：优先开源组件，支持单机/局域网/云服务器部署。
6. **离线容错**：车间网络不稳定时，移动端支持离线报工，联网后自动同步。

#### 1.4.3 避坑指南：这些"免费"其实有坑

| 组件 | 陷阱 | 正确做法 |
|------|------|----------|
| MySQL Community | GPL 协议，若分发二进制可能触发传染 | 改用 **MariaDB**（GPL仅限服务端，不传染客户端）或 **PostgreSQL**（类BSD） |
| MinIO | AGPL 协议，通过网络提供服务时需开源全部代码 | 改用 **本地磁盘存储**，10人厂无需对象存储 |
| Docker Desktop | 企业使用（250人+或1000万+营收）需付费订阅 | 改用 **Docker Engine（免费）** 或直接 **JAR 包部署**，不用 Docker |
| LODOP | 高级功能需购买授权 | 改用 **浏览器原生打印** + CSS @media print |
| 短信通知 | 每条 0.03~0.05 元，长期累积 | 改用 **WebSocket 站内推送** + **企业微信 Webhook（免费）** |
| 微信小程序 | 认证费 300 元/年 | 改用 **H5 网页**，添加到手机桌面即可，体验接近原生 |
| 阿里云 OSS/S3 | 按量计费，长期有成本 | 改用 **本地磁盘** + 自动备份脚本 |
| 商业图表库（Highcharts 等） | 商用需授权 | 坚持用 **ECharts**（Apache-2.0，完全免费） |
| Navicat | 商业软件，需购买授权 | 改用 **DBeaver**（开源，ASL-2.0）或 **HeidiSQL**（GPL，免费） |

### 3.2 部署形态
- **单机版（推荐，零费用）**：一台工厂现有 Windows PC（i5/8G/256G SSD 即可），内置 MariaDB + Redis + 后端 JAR + 前端静态文件，局域网访问，**无需购买任何服务器**。

- **混合版（可选）**：本地部署核心生产数据，云端只做数据备份与老板看板同步。

### 5.2 核心业务流程

#### 5.2.1 销售 → 生产 → 回款主流程

```
客户询价 → 销售报价 → 客户下单 → 销售订单 → MRP 拆分工单
    → 生产排程 → 原料采购入库 → 工单领料出库 → 注塑生产 → 质检
    → 合格品成品入库 / 不良品转入不良品仓
    → 发货 → 销售出库 → 客户签收 → 开票 → 回款登记 → 对账
```

#### 5.2.2 生产报工流程

```
开工：扫描机台二维码 / 选择工单 → 选择操作工 → 选择模具 → 开始生产
报产：输入班次产量（自动 / 模次 × 穴数） → 系统记录工时
质检：首件合格 → 连续生产；巡检不良 → 停机调整
完工：输入末件数 → 产品入库 → 工单关闭
```

#### 5.2.3 计件工资计算

```
员工日工资 = Σ(当日各工单合格品数量 × 对应工序计件单价) + 补贴 - 扣款
月汇总 = 每日工资累加 + 全勤奖 + 绩效 - 社保/个税（可选）
```

### 5.3 仓库管理详细设计

#### 5.3.1 仓库与库位结构

```
工厂
├── 原料仓（RAW）
│   ├── A 区（原料区）
│   │   ├── A-01-01（货架-层-位）
│   │   └── A-01-02
│   └── B 区（色母区）
├── 半成品仓（SEMI）
│   └── C 区
├── 成品仓（FINISH）
│   └── D 区
└── 不良品仓（DEFECT）
    └── E 区
```

#### 5.3.2 库存流转业务规则

| 业务场景 | 单据 | 仓库变化 | 影响库存 |
|----------|------|----------|----------|
| 原料采购到货 | 采购入库单 | 原料仓 + | stock.qty + |
| 工单开工领料 | 领料出库单 | 原料仓 - | stock.qty - / locked_qty - |
| 工单生产剩余退料 | 退料入库单 | 原料仓 + | stock.qty + |
| 生产完工 | 成品入库单 | 成品仓 + | stock.qty + |
| 销售发货 | 销售出库单 | 成品仓 - | stock.qty - |
| 不良品隔离 | 不良品入库单 | 成品仓 - / 不良品仓 + | stock.qty - |
| 仓库间调拨 | 调拨单 | 出库仓 - / 入库仓 + | stock.qty -/+ |
| 盘点盈亏 | 盘点单 | 对应仓库 +/- | stock.qty +/- |

#### 5.3.3 批次管理规则

1. **原料批次**：按采购批次生成，格式 `R-YYYYMMDD-001`，先进先出（FIFO）。
2. **成品批次**：按生产日期生成，格式 `P-YYYYMMDD-机台号-模次`，支持按批次追溯。
3. **色母/辅料批次**：单独管理，按开封日期记录有效期。
4. **不良品批次**：转入不良品仓时强制记录不良原因与处理意见。

#### 5.3.4 库存锁定机制

```
工单创建 → 根据 BOM/原料用量计算需求 → 锁定库存（locked_qty +）
工单领料 → 扣减实际库存（qty -）并释放锁定（locked_qty -）
工单取消/关闭 → 释放全部锁定（locked_qty -）
```

#### 5.3.5 库存预警规则

| 预警类型 | 触发条件 | 处理建议 |
|----------|----------|----------|
| 安全库存预警 | 可用库存 < safe_stock | 提示采购或调整生产计划 |
| 呆滞料预警 | 原料超过 90 天未使用 | 提示清理或降级使用 |
| 超储预警 | 成品库存 > 月销量 2 倍 | 提示控制生产节奏 |
| 效期预警 | 原料批次临近保质期（30 天内） | 提示优先使用 |

#### 5.3.6 盘点流程

```
创建盘点单 → 冻结盘点区域库存 → 现场扫码/手工录入实盘数量
    → 生成盈亏差异 → 审核 → 调整库存 → 解除冻结
```

## 8. 状态机与单据流转

### 8.1 销售订单状态流转

```
DRAFT(草稿) ──确认──→ CONFIRMED(已确认) ──生成工单──→ PRODUCING(生产中)
    │                      │                        │
    │                      │──取消──→ CANCELLED     │──全部发货──→ DELIVERED(已发货)
    │                      │                        │
    └──────────────────────┴────────────────────────┴──关闭──→ CLOSED(已关闭)
```

### 8.2 生产工单状态流转

```
WAITING(待排程) ──排程──→ SCHEDULED(已排程) ──开工──→ RUNNING(生产中)
    │                                            │    │
    │                                            │    └──暂停──→ PAUSED(已暂停)──恢复──→ RUNNING
    │                                            │
    │                                            └──完工──→ FINISHED(已完工)──关闭──→ CLOSED(已关闭)
```

### 8.3 质检结果处理

```
PASS(合格) ──→ 产品入库
FAIL(不合格) ──→ 记录不良 → 判定处理方式：
    ├── 返工 ──→ 重新报工 ──→ 返工质检
    ├── 让步接收 ──→ 标记后入库
    └── 报废 ──→ 扣减库存 ──→ 计入不良损失
CONDITIONAL(条件接收) ──→ 老板确认 ──→ 让步入库
```

### 8.4 仓库单据状态流转

#### 8.4.1 调拨单
```
DRAFT(草稿) ──确认发货──→ SHIPPED(已发货) ──收货确认──→ RECEIVED(已收货) ──关闭──→ CLOSED(已关闭)
```

#### 8.4.2 盘点单
```
DRAFT(草稿) ──开始盘点──→ COUNTING(盘点中) ──录入完成──→ PENDING_APPROVE(待审核)
    │                                                          │
    │──取消──→ CANCELLED(已取消)                              ├──审核通过──→ FINISHED(已完成)
                                                              │
                                                              └──驳回──→ COUNTING(重新盘点)
```

## 9. 计件工资详细算法

### 9.1 日工资计算流程

```
1. 取当日该员工所有报工记录（prod_report）
2. 按工单分组，汇总每个工单的合格品数量 = qty - bad_qty
3. 查询对应产品的当前计件单价（piece_price，取生效日期最近的）
4. 计算计件工资 = Σ(合格品数量 × 计件单价)
5. 加上当日补贴/扣款（salary_adjust）
6. 写入 salary_daily
```

### 9.2 计算公式

```
日工资 = Σ(工单合格品数_i × 产品计件单价_i) + Σ(当日补贴) - Σ(当日扣款)

其中：
  合格品数 = 报工产量 - 不良数量
  计件单价 = piece_price 中 effective_date <= 当日 AND (expire_date IS NULL OR expire_date > 当日) 的最新记录

月工资 = Σ(当月每日日工资) + 月度奖惩 + 全勤奖 - 社保代扣 - 个税
```

### 9.3 边界条件
- **跨班次工单**：同一工单日夜班分别报工，按班次分别计算。
- **返工品**：返工合格后重新报工，按新报工日期计入当日工资，不重复扣减。
- **单价变更**：以报工日期对应的生效单价为准，历史数据不回溯。
- **停机期间**：停机时间不计入工时，不影响计件工资。
- **多人同机台**：同一机台同一班次可能有多人，按各自报工数量分别计算。

## 11. 看板指标定义

### 11.1 老板驾驶舱

| 指标 | 计算公式 | 刷新频率 |
|------|----------|----------|
| 当月订单总额 | Σ(当月确认的 sale_order.total_amount) | 实时 |
| 当月回款总额 | Σ(当月 payment_record.pay_amount) | 实时 |
| 应收账款余额 | 订单总额 - 已回款总额 | 实时 |
| 回款率 | 已回款 / 订单总额 × 100% | 实时 |
| 当月完工数量 | Σ(当月完工工单的 qualified_qty) | 实时 |
| 当月不良率 | 总不良数 / 总产量 × 100% | 实时 |
| 当月工资总额 | Σ(当月 salary_daily.total_amount) | 日结 |
| 当月费用总额 | Σ(当月 expense_record.amount) | 实时 |
| 当月毛利润 | 回款总额 - 原料成本 - 工资 - 费用 | 日结 |
| OEE（设备综合效率） | 时间开动率 × 性能开动率 × 合格品率 | 实时 |

### 11.2 OEE 计算公式

```
时间开动率 = 实际运行时间 / 计划运行时间
  计划运行时间 = 班次总时长 - 计划停机（换模、保养）
  实际运行时间 = 计划运行时间 - 非计划停机

性能开动率 = 实际产量 / 理论产量
  理论产量 = 实际运行时间(秒) / 标准周期(秒) × 穴数

合格品率 = 合格品数量 / 总产量 × 100%

OEE = 时间开动率 × 性能开动率 × 合格品率
```

## 19. 安全设计

1. **登录安全**：密码使用 BCrypt 加密；失败 5 次锁定 15 分钟。
2. **权限控制**：RBAC，按角色分配菜单与接口权限。
3. **数据隔离**：多租户字段预留，当前单厂可简化；销售、工资等敏感数据仅老板/财务可见。
4. **操作日志**：关键业务操作（改单、删单、改工资）记录日志，含修改前后值。
5. **接口防重**：报工等关键接口基于唯一键防重复提交。
6. **SQL 防注入**：MyBatis-Plus 参数化查询，禁止拼接 SQL。
7. **XSS 防护**：前端输入转义，后端过滤 HTML 标签。
8. **HTTPS**：生产环境强制 HTTPS。
9. **备份策略**：每日自动全量备份 MySQL，保留 30 天。

### 25.2 备份方案（零费用）

```
方案一：Windows 计划任务（推荐）
  1. 每日凌晨 2:00 自动执行 backup.bat
  2. mysqldump 全量备份 → 压缩 → 保留 30 天
  3. 备份文件存到 D:\backup\ 目录
  4. 可选：复制到 U 盘/移动硬盘（物理隔离）

方案二：批处理脚本示例
  @echo off
  set DATE=%date:~0,4%%date:~5,2%%date:~8,2%
  set BACKUP_DIR=D:\backup\%DATE%
  mkdir %BACKUP_DIR%
  "C:\Program Files\MariaDB 10.11\bin\mysqldump" -u root -pinject@2026 inject_erp > %BACKUP_DIR%\inject_erp.sql
  "C:\Program Files\7-Zip\7z.exe" a %BACKUP_DIR%\inject_erp.7z %BACKUP_DIR%\inject_erp.sql
  del %BACKUP_DIR%\inject_erp.sql
  forfiles /p "D:\backup" /d -30 /c "cmd /c rd /s /q @path"
```
