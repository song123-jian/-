# 权限矩阵

使用 Skill：requirements-doc

## 1. 角色定义

| 角色 | 定位 |
| --- | --- |
| 老板 | 查看经营驾驶舱、财务汇总、生产质量库存指标 |
| 管理员 | 系统配置、用户、基础资料、全部业务模块和运维页面 |
| 生产主管 | 生产订单、报工、领料、成品入库、生产看板、预警 |
| 仓库员 | 库存查询、库存台账、入库、出库、调拨、盘点 |
| 质检员 | 质检录入、质检记录、不良分析、质量看板 |
| 财务 | 回款、费用、对账单、工资汇总、财务报表 |
| 操作员 | 移动端报工、质检、盘点、调拨接收和消息查看 |

## 3. 数据权限原则

1. 前端菜单权限用于体验控制，不能作为唯一安全边界。
2. Supabase 生产项目必须结合 RLS 或等价权限控制限制数据访问。
3. 管理员可维护全量数据，普通角色只访问授权模块和本人相关数据。
4. 移动端个人产量、工资和消息仅允许查看本人数据。
5. 所有关键增删改必须写入 `sys_operation_log`。

## 4. RLS 验证清单

- [ ] 未登录请求不能读取业务表。
- [ ] 操作员不能读取其他员工工资。
- [ ] 仓库员不能修改系统配置。
- [ ] 财务不能绕过业务校验直接修改库存。
- [ ] 质检员不能修改销售订单金额。
- [ ] 管理员关键操作写入操作日志。

## 5. 当前说明

当前代码和数据库已具备角色字段、登录 RPC、操作日志和前端路由守卫。生产上线前仍需在目标 Supabase 项目中完成 RLS 策略落地和验证截图归档。
