## 3. 关键约束

- 编码类字段大多具备唯一约束，例如产品、客户、供应商、仓库、机台、模具、订单号。
- `stock` 使用产品、仓库、库位、批次组合唯一约束。
- `salary_daily` 使用用户和日期唯一约束。
- 金额字段采用 decimal 类型，前端工具函数负责格式化和汇总。
- 注塑专业编码类字段具备唯一约束，例如 `card_no`、`trial_no`、`mix_no`、`trace_no`、`check_no`、`order_no`、`plan_no`、`event_no`、`template_no`、`complaint_no`、`change_no`、`requisition_no`。
- `material_mix_order.regrind_ratio` 使用数据库检查约束限制在 0-100。
- 注塑专业状态字段统一通过前端模块定义和 Supabase 适配层写入，便于管理端和移动端共用。

## 4. 数据治理要求

1. 历史单据关联的基础资料不得硬删除。
2. 库存变动必须同时形成 `stock_move` 台账。
3. 工资、回款、费用、对账金额必须可追溯到源单。
4. 关键单据修改必须写入操作日志。
5. 注塑工单开工前必须保留工艺卡、首件确认和齐套检查三类证据。
6. 原料批次、配料、报工、质检、成品入库、销售出库和客诉应通过 `batch_trace_link` 保留追溯链。
