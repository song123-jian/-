# 管理员运维手册

使用 Skill：requirements-doc、runtime-diagnostics

## 1. 日常巡检

| 项目 | 频率 | 检查点 |
| --- | --- | --- |
| 登录可用性 | 每日 | 管理端、移动端可登录 |
| 数据库连接 | 每日 | 工作台、库存、系统配置可读取 |
| 操作日志 | 每日 | 关键新增、修改、删除记录完整 |
| 备份状态 | 每日 | 数据库和 Storage 备份完成 |
| 业务预警 | 每日 | 低库存、模具保养、交期、质量预警 |
| 构建产物 | 每次发布 | 管理端和移动端产物版本一致 |

## 2. 常用命令

```bash
npm run check-encoding
npm test
npm run build
npm run dev:admin
npm run dev:mobile
```

## 3. 账号维护

1. 管理员在基础资料/用户管理中创建用户。
2. 分配角色、状态、手机号和初始密码。
3. 用户首次登录后应修改密码。
4. 离职或停用人员应设置为禁用，不直接删除历史业务关联。

## 4. 配置维护

系统管理/系统配置用于维护：

- 公司名称
- 备份策略
- 库存预警阈值
- 模具保养周期
- 业务参数

配置变更后必须检查操作日志。

## 5. 故障处理

| 现象 | 排查步骤 |
| --- | --- |
| 页面提示未配置 Supabase | 检查 `.env.local` 或部署变量 |
| 登录失败 | 检查 Supabase Auth、`sys_user`、`erp_login` RPC |
| 库存不一致 | 查询 `stock`、`stock_move`、采购入库、生产领料、成品入库、销售出库单据 |
| 工资异常 | 查询 `prod_report`、`piece_price`、`salary_daily`、`salary_adjust` |
| 文件上传失败 | 检查 `erp-files` bucket、Storage 策略和文件大小类型 |
| 构建失败 | 执行对应前端 `npm --prefix <dir> run build` 定位错误 |

## 6. 发布流程

1. 拉取目标版本。
2. 执行全量验证命令。
3. 备份数据库和 Storage。
4. 构建并发布管理端、移动端。
5. 执行核心业务烟测。
6. 更新版本与发布说明。

## 7. 回滚入口

发布失败按 [上线回滚方案.md](上线回滚方案.md) 执行。
