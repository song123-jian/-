# 备份与恢复说明

使用 Skill：requirements-doc、project-plan

## 1. 目标

确保生产环境出现误操作、发布失败、数据损坏或云服务异常时，可以按 RPO 24 小时、RTO 4 小时的目标恢复核心业务。

## 2. 备份范围

| 对象 | 内容 | 频率 |
| --- | --- | --- |
| 数据库 | Supabase PostgreSQL 全量备份 | 每日一次，上线前手动一次 |
| Storage | `erp-files` bucket 文件 | 每日一次或随数据库备份同步 |
| 前端产物 | `frontend-admin/dist`、`frontend-mobile/dist` | 每次发布留存 |
| 环境变量 | 生产变量名称和值的安全备份 | 每次变更留存 |
| 数据库脚本 | `database/init.sql`、`database/supabase-cloud.sql` 和迁移脚本 | 每次变更留存 |

## 3. 备份步骤

1. 在 Supabase 控制台确认当前项目无长事务和批量导入任务。
2. 执行数据库备份或导出。
3. 导出 Storage bucket 文件清单和对象。
4. 记录备份时间、操作者、备份范围、存储位置和校验结果。
5. 将备份记录写入运维台账。

## 4. 恢复步骤

1. 暂停生产写入入口或通知业务停止操作。
2. 确认恢复点和影响范围。
3. 在隔离环境恢复备份，验证登录、库存、订单、工资、财务和报表。
4. 确认恢复数据无误后切换生产或执行生产恢复。
5. 恢复后执行 20 条核心路径中的高风险路径：登录、库存、采购入库、生产领料、报工、销售出库、回款、工资、对账。
6. 记录恢复结果和遗留问题。

## 6. 硬性上线要求

生产上线前必须至少完成一次备份恢复桌面推演；正式生产切换前必须完成一次数据库备份。
